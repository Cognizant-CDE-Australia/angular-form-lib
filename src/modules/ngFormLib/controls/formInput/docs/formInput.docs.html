Form Input FormInput.js

ngFormLib.controls.formInput

The Form Input directive acts like a macro in that it expands into a <label><input> and some auxillary HTML elements, but does change the $scope of those elements.

It works this way because the elements are inter-related. For example, when the required attribute is set to true the <label> element is changed so that a required-indicator-CSS-class is applied to the element.

Plugin dependency

Form Input requires the form controls service module, the field error controller module , the error message container module to be loaded.

Examples

Use form-input to create a complete HTML structure containing a label and select element that reacts to the error state of the field.

Live demo

Usage

Add the form-input directive to an element.

Options

Options can be passed via attributes on the directive.

Name Type Default Description
field-hint string Text that is related to the field but is not the field's label. Often used to provide additional information about the field.
input-prefix string Creates a Bootstrap input group around the input element and displays the input-prefix string inside a <span class="input-group-addon"></span> element, which appears before the input element.
input-suffix string Creates a Bootstrap input group around the input element and displays the input-suffix string inside a <span class="input-group-addon"></span> element, which appears after the input element.
placeholder string Placeholder text that appears inside the input element when there is no existing value. Alias for ff-placeholder.
template string See Form Controls Service options The HTML template to use for the control. The default value can be overridden on the element, or globally by setting the default value in the Form Controls Service.

Unlike the other form control templates, this attribute is a STRING, not a URL! The reason for this is that the input type="" attribute is a write-once attribute. To avoid having to create separate templates for the different input types (e.g.: text|time|number|tel|...), it is easier to manipulate a template string to insert the type attribute into the template BEFORE it is compiled.
